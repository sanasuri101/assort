# 06-03: Knowledge Base & Settings

**Goal**: Allow providers to manage their Knowledge Base (FAQ) and configure practice settings (hours, prompts).

**Depends on**: 06-01, Phase 4 (KB implementation)

## Requirements
- **KB Management**: List existing KB entries. Add/Edit/Delete entries.
- **Settings**: Update Practice Name, Office Hours, Insurance Plans.

## Implementation Details

### 1. Backend API (`app/api/dashboard.py`)
- `GET /api/knowledge`: List all Q&A.
- `POST /api/knowledge`: Add/Update Q&A (re-embeds using OpenAI).
- `DELETE /api/knowledge/{id}`: Remove.
- `GET /api/settings`: Get current config.
- `POST /api/settings`: Update config (persist to Redis/JSON).

### 2. Frontend Components
- `pages/Knowledge.tsx`: Table with Search. Dialog for Add/Edit.
- `pages/Settings.tsx`: Form for practice details.

## Verification
- **Manual**: Add a new FAQ "What color is the wall?". Call the bot and ask "What color is the wall?". Verify it answers correctly using new data.
